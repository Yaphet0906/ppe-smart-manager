## 重大决策记录

### 决策1: 项目暂停/转型（HSE承诺书系统）
**时间**: 2026-03-27 08:00
**背景**: HSE年度责任承诺书系统因邮件发送通道问题无法实施
**决策**: 
- 暂停HSE年度责任承诺书系统开发
- 原因：Outlook企业邮箱无法接入，邮件发送功能受阻
- 结论：该项目不能作为独立软件，只能作为学习平台的附件

**影响**: 避免投入无法落地的项目，节省开发资源

---

### 决策2: PPESmartManager项目可行性确认
**时间**: 2026-03-27 08:15
**背景**: 评估PPE库存管理系统是否值得继续
**决策**:
- 继续推进PPESmartManager项目
- 原因：
  1. 技术栈成熟（Vue3 + Node.js + MySQL）
  2. 不依赖外部邮件通道
  3. 飞书企业渗透率仅11%，市场空间大
  4. 目标用户明确（工厂HSE人员）

**影响**: 确定项目方向，集中资源开发

---

### 决策3: 产品定位转变（从专业管理到懒人工具）
**时间**: 2026-03-27 08:17
**背景**: 讨论目标客户的真实需求
**决策**:
- 核心定位：**让人偷懒**
- 不是"专业管理"，而是"省人力"
- 目标客户：没有专职库管的工厂
- 核心价值：
  1. 员工自助领用（扫码，无需库管）
  2. 自动记录（无需手工录入Excel）
  3. 一键报表（HSE月底不加班）

**关键洞察**:
> "只要他们想懒惰，我们就能把这些东西给他们用"
> —— 来自大哥的洞察

**影响**: 彻底转变产品设计思路，从复杂功能转向极简流程

---

### 决策4: SaaS模式确定
**时间**: 2026-03-27 08:21
**背景**: 讨论部署方式（内网 vs SaaS）
**决策**:
- 采用SaaS模式（阿里云服务器 + 多租户）
- 原因：
  1. 不需要客户IT审批
  2. 开箱即用
  3. 订阅制收费
- 多租户数据隔离：数据库添加company_id字段

**影响**: 确定商业模式和技术架构

---

### 决策5: 入库方式优化（截图OCR）
**时间**: 2026-03-27 08:25
**背景**: 讨论手机端Excel导入的不便
**决策**:
- 优先使用**截图OCR入库**而非Excel导入
- 原因（手机端场景）：
  1. 截图比下载Excel更快
  2. 不需要在文件管理器中找文件
  3. 30秒完成 vs 2-3分钟
- 技术方案：讯飞OCR识别邮件/订单截图

**影响**: 确定核心差异化功能

---

### 决策6: 扫码领用流程简化
**时间**: 2026-03-27 08:30
**背景**: 讨论员工领用流程
**决策**:
- **无需登录**，填姓名+手机号即可领用
- 原因：
  1. 降低使用门槛
  2. 仓库有物理隔离（钥匙管理），安全性足够
  3. 姓名+手机号双重验证，准确率够用
- 流程：扫码 → 填姓名/手机号 → 选物品 → 提交

**影响**: 极大简化员工端使用流程

---

### 决策7: 首次登录强制改密码
**时间**: 2026-03-27 14:10
**背景**: 安全策略讨论
**决策**:
- 初始密码可用简单密码（如123456!）
- 但**首次登录必须强制修改密码**
- 技术实现：
  1. 数据库添加is_first_login字段
  2. 登录接口返回首次登录状态
  3. 前端跳转修改密码页面

**影响**: 平衡部署便利性和安全性

## 技术架构决策

### 前端技术栈
- **框架**: Vue 3
- **UI库**: Element Plus
- **路由**: Vue Router
- **状态管理**: Pinia
- **HTTP客户端**: Axios

### 后端技术栈
- **运行时**: Node.js 18+
- **框架**: Express
- **数据库**: MySQL 8.0
- **认证**: JWT
- **OCR**: 讯飞开放平台

### 部署架构
- **前端**: GitHub Pages
- **后端**: 阿里云服务器（待部署）
- **数据库**: 阿里云RDS或自建MySQL
- **CI/CD**: GitHub Actions
