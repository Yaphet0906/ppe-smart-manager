**涉及文件**:

### 任务 4: 不同类别不同尺码属性
**优先级**: ⭐⭐⭐⭐  
**描述**: 根据物品类别，显示对应的尺码类型  

**类别与尺码映射**:
| 类别 | 尺码选项 |
|------|---------|
| 安全鞋 | 36码、37码、38码...46码 |
| 工作服 | S、M、L、XL、XXL、XXXL |
| 手套 | S、M、L、XL |
| 安全帽 | 52-56cm、57-62cm |
| 口罩 | 均码 / 成人口罩、儿童口罩 |

**功能需求**:
1. 添加物品时，选择类别后自动显示对应的尺码选项
2. 尺码作为独立字段存储
3. 同一物品（如安全鞋）不同尺码作为独立库存项管理

**涉及文件**:
- 数据库: 可能需要 `item_specs` 规格表 或 `size` 字段
- `frontend/src/views/PPEList.vue` - 添加/编辑表单
- `backend/routes/ppe.js` - 接口支持尺码字段

## 🗄️ 数据库变更计划

### 方案 A: 简单方案（推荐明天先用）
在 `inv_items` 表添加 `size` 字段：
```sql
ALTER TABLE inv_items ADD COLUMN size VARCHAR(20) COMMENT '尺码/规格' AFTER model;
```

每个尺码作为独立物品：
- 安全鞋-42码（id=1, name=安全鞋, size=42码）
- 安全鞋-43码（id=2, name=安全鞋, size=43码）

### 方案 B: 规范化方案（后期优化）
拆分 `inv_items` 为：
- `inv_item_templates` - 物品模板（安全鞋、手套等）
- `inv_item_specs` - 规格表（42码、M码等）
- `inv_items` - 实际库存物品（关联 template + spec）

**明天建议用方案A**，简单快速。
