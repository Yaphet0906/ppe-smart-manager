# PPE智能管理系统 v1.5 发布说明

**发布日期**: 2026-04-03  
**版本号**: v1.5  
**代号**: 精简入库 (Streamlined Inbound)

---

## 🎯 核心改进

### 1. 入库流程重构
- **整合截图入库**: 将截图入库功能整合到入库管理模块
- **简化手动入库**: 去除模式切换，直接填写完整用品信息入库
- **日期格式化**: 入库记录日期显示为 `2026-04-02` 格式

### 2. 软删除机制
- **删除留痕**: 删除用品时强制要求填写删除原因
- **数据保留**: 使用软删除，数据不真正删除，仅标记 deleted_at
- **删除日志**: 新增 delete_logs 表，记录删除人、删除时间、删除原因和完整数据备份

### 3. 库存管理优化
- **移除编辑功能**: 库存数量只能通过入库/出库流程变更，不允许直接修改
- **表格布局优化**: 展开后的尺码明细独立成列显示
- **精简操作**: 只保留删除按钮，编辑按钮已移除

---

## 📁 文件变更

### 后端
- `backend/routes/ppe.js` - 完善删除接口，支持软删除和日志记录

### 前端
- `frontend/src/views/PPEInbound.vue` - 重构入库模块，整合截图和手动入库
- `frontend/src/views/PPEList.vue` - 移除编辑功能，优化表格布局
- `frontend/src/views/Layout.vue` - 移除独立的"截图入库"菜单

### 数据库
- `data/delete_logs.sql` - 新增删除日志表

---

## 🚀 部署说明

### 1. 数据库更新
```bash
mysql -u root -p ppe_smart_manager < data/delete_logs.sql
```

### 2. 前端构建
```bash
npm run build
```

### 3. 重启服务
```bash
# 后端
pm2 restart ppe-backend

# 或手动
npm run serve
```
